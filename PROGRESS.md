Старые gzserver/ноды оставались висеть в фоне и держали DDS shared-memory порты и порт Gazebo-сервера, из-за чего новый запуск падал (`RTPS_TRANSPORT_SHM` ошибки, `gzserver exit code 255`). - **Правило на будущее: `ros2 launch` останавливать только `Ctrl+C`, никогда `Ctrl+Z`**

## Шпаргалка: как самому дебажить ROS2, когда "не работает"
Правило: не гадать, а идти по цепочке данных от источника к получателю и смотреть, где обрыв.

- `ros2 node list` — какие ноды реально сейчас запущены
- `ros2 topic list` — какие топики вообще существуют
- `ros2 topic echo /имя_топика` — САМАЯ полезная команда: смотришь, реально ли что-то публикуется в топик прямо сейчас
- `ros2 topic hz /имя_топика` — с какой частотой идут сообщения (если 0 — публикатора нет или он молчит)
- `ros2 topic info /имя_топика --verbose` — кто publisher, кто subscriber у топика (находишь, где в цепочке нода "потерялась")
- `ros2 run rqt_graph rqt_graph` — визуальный граф нод/топиков, если словами уже сложно представить
- `ros2 param get /имя_ноды имя_параметра` — посмотреть текущее значение параметра ноды (пригодится для `danger_distance_m` и т.п.)

## Полезные команды сборки/запуска (шпаргалка на каждый день)
Внимание: на машине есть ДВА каталога `ros2_ws` — старый `~/ros2_ws` (не наш, там `hackathon_digital_twin`) и наш `~/gazon/ros2_ws`. Всегда явно `cd ~/gazon/ros2_ws`, не просто `cd ros2_ws` откуда попало — иначе получите `Package not found`.
```bash
cd ~/gazon && git pull
cd ~/gazon/ros2_ws
colcon build --packages-select <имя_пакета>   # или без --packages-select чтобы пересобрать всё
source install/setup.bash
ros2 launch mower_description gazebo_safety_demo.launch.py        # с GUI
ros2 launch mower_description gazebo_safety_demo.launch.py gui:=false   # без GUI, если тормозит
```
